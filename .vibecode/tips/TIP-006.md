## ROOT CAUSE (đọc kỹ)
```
[User click menu /hcc]
    │
    ▼
dcc.Location pathname = "/hcc"
    │
    ├──► render_page("/hcc") → Trả về layout HCC ✅
    │
```

F5 không lỗi vì callbacks chỉ trigger khi có thay đổi Input — F5 không thay đổi tabs-navigation.

## TASK

### Bước 1: Sửa `dash_app/app.py` L245
```python
# TRƯỚC:
    return "tab-kpi"  # Default

# SAU:
    return None  # Không trigger BCCP callbacks khi ở trang khác
```

### Bước 2: Sửa tất cả callbacks BCCP — thêm check `tab_val is None`

Grep tất cả callbacks: `grep -n "tab_val != " dash_app/callbacks/*.py`

Với mỗi file tìm thấy pattern `if tab_val != "tab-xxx":`, sửa thành:
```python
# TRƯỚC:
    if tab_val != "tab-kpi":
        return [dash.no_update] * N

# SAU:
    if tab_val != "tab-kpi" or tab_val is None:
        return [dash.no_update] * N
```

### Bước 3: Kiểm tra `dash_app/app.py` callback `update_header_subtitle`
Callback này trong `sidebar_callbacks.py` có thể cũng bị ảnh hưởng. Kiểm tra xem có `Input("tabs-navigation", ...)` không — nếu có thì thêm guard tương tự.

## SPECIFICATIONS
- `sync_url_to_tabs_navigation` chỉ trả về string tab-xxx khi URL là `/bccp*`
- Mọi URL khác (`/`, `/hcc`, `/tcbc`, `/ppbl`, `/import`) → return `None`
- Tất cả callbacks BCCP: `tab_val is None` → `return [dash.no_update] * N` (không crash, không render)
